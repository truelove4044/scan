# 台灣身分證掃描 Web App 規劃

## 產品目標與使用情境

本 Web App 讓使用者透過手機或桌機瀏覽器相機拍攝台灣身分證正面，系統自動辨識可見欄位，使用者確認或修正後再送出資料。第一版重點是降低人工輸入成本，同時保留人工確認機制，避免 OCR 誤判直接進入後續流程。

主要使用情境：

- 使用者在註冊、身分核驗或資料建檔流程中，需要快速填入身分證正面資訊。
- 使用者可使用行動裝置相機即時拍攝，也可在桌機瀏覽器使用外接或內建相機。
- 系統只處理台灣身分證正面，不包含健保卡、駕照、居留證或護照。

## MVP 功能範圍

第一版包含：

- 瀏覽器相機啟用與權限處理。
- 身分證正面取景框。
- 拍照、重拍與送出辨識。
- 前端照片裁切、壓縮與基本畫質檢查。
- 後端 OCR 呼叫、欄位解析與格式驗證。
- 辨識結果確認頁，讓使用者修正欄位。
- 確認後送出結構化資料給業務系統。

第一版不包含：

- 原生 iOS 或 Android App。
- 身分證背面辨識。
- 文件真偽鑑別。
- 活體偵測。
- 預設保存原始影像。
- 多證件類型切換。

## 使用流程

1. 使用者進入身分證掃描頁。
2. 系統請求相機權限。
3. 使用者將身分證正面對齊取景框。
4. 系統進行基本畫質提示，例如過暗、模糊、反光或未對齊。
5. 使用者拍照。
6. 使用者可選擇重拍或送出辨識。
7. 後端接收影像，呼叫 OCR 引擎並解析欄位。
8. 前端顯示辨識結果與欄位信心狀態。
9. 使用者確認或修正欄位。
10. 使用者送出確認後的結構化資料。
11. 系統回到原業務流程的下一步。

## UI 頁面規劃

### 掃描頁

- 顯示相機預覽。
- 顯示身分證比例取景框。
- 提供拍照按鈕。
- 顯示相機權限、鏡頭不可用、裝置不支援時的使用者可理解訊息。
- 在畫質不佳時顯示簡短提示。

### 預覽頁

- 顯示拍攝後的身分證影像預覽。
- 提供重拍與送出辨識按鈕。
- 若前端畫質檢查未通過，提示使用者重新拍攝。

### 確認頁

- 顯示可編輯欄位：
  - 姓名
  - 身分證字號
  - 出生日期
  - 發證日期
  - 發證地點
  - 其他正面可見且業務需要的欄位
- 對低信心或驗證未通過欄位做視覺標示。
- 提供返回重拍與確認送出按鈕。
- 不顯示 API 欄位名、狀態碼、debug 資訊或 OCR 原始輸出。

### 結果回饋

- 送出成功後回到業務流程下一步。
- 送出失敗時提示使用者稍後再試或重新確認資料。
- 錯誤訊息使用一般使用者可理解語意，不顯示技術細節。

## OCR 與欄位解析設計

前端責任：

- 使用 `getUserMedia` 啟用相機。
- 依取景框裁切影像。
- 壓縮影像至後端可接受大小。
- 進行基本畫質檢查：
  - 亮度是否過低。
  - 影像是否明顯模糊。
  - 身分證是否大致位於取景框內。
- 將影像以安全傳輸方式送至後端。

後端責任：

- 接收影像並做請求大小限制。
- 呼叫 OCR Provider 介面。
- 將 OCR 原始結果轉為欄位候選值。
- 根據台灣身分證格式與日期格式做驗證。
- 回傳結構化辨識結果、欄位信心與可修正提示。
- 預設不持久化原始影像。

OCR Provider 設計為可替換介面，不在第一版綁定單一供應商。介面輸入為影像檔，輸出為文字區塊、座標與信心分數。欄位解析層只依賴抽象 OCR 結果，避免後續更換服務時影響前端與業務流程。

## 台灣身分證資料驗證

身分證字號驗證：

- 格式為 1 個英文字母加 9 個數字。
- 第一碼英文字母需轉為對應數值。
- 第二碼為性別碼，第一版接受 `1` 與 `2`。
- 檢查碼需符合台灣身分證字號規則。

日期驗證：

- 支援民國年月日格式辨識。
- 轉換為使用者可理解的日期顯示。
- 後端可另外保存標準化日期格式，供業務系統使用。
- 無法可靠解析時，標示為需使用者確認。

欄位完整性：

- 姓名與身分證字號為必要欄位。
- 出生日期、發證日期與發證地點依業務流程決定是否必填。
- 任一必要欄位未通過驗證時，不允許直接送出，需使用者修正或重拍。

## 資料安全與隱私設計

- 全程使用 HTTPS。
- 原始影像只用於當次 OCR 處理，預設不保存。
- 後端處理完成後清除暫存影像。
- 只回傳業務所需欄位，不回傳 OCR 原始全文給前端。
- 日誌不得記錄完整身分證字號、姓名或原始影像。
- 若未來需要保存影像，必須新增明確授權、保存期限、刪除機制與稽核紀錄。
- 欄位送出前必須經使用者確認。

## 技術架構

建議架構：

- Frontend：負責相機、取景框、拍照、預覽、確認表單與使用者互動。
- Backend API：負責接收影像、呼叫 OCR、欄位解析、驗證與資料送出。
- OCR Provider：可替換的外部或內部 OCR 引擎。
- Business System：接收使用者確認後的身分證資料。

資料流：

1. Frontend 拍攝並裁切影像。
2. Frontend 呼叫 `POST /api/id-card/recognize`。
3. Backend 呼叫 OCR Provider。
4. Backend 解析並驗證欄位。
5. Frontend 顯示確認表單。
6. 使用者修正後呼叫 `POST /api/id-card/confirm`。
7. Backend 將確認資料送入業務系統。

## API / 資料結構草案

### `POST /api/id-card/recognize`

用途：接收身分證正面影像並回傳辨識結果。

Request：

```json
{
  "image": "base64-encoded-image",
  "imageMimeType": "image/jpeg"
}
```

Response：

```json
{
  "recognizedFields": {
    "name": {
      "value": "王小明",
      "confidence": 0.95,
      "needsReview": false
    },
    "nationalId": {
      "value": "A123456789",
      "confidence": 0.92,
      "needsReview": false
    },
    "birthDate": {
      "value": "1990-01-01",
      "displayValue": "79年1月1日",
      "confidence": 0.84,
      "needsReview": true
    },
    "issueDate": {
      "value": "2020-05-20",
      "displayValue": "109年5月20日",
      "confidence": 0.81,
      "needsReview": true
    },
    "issueLocation": {
      "value": "臺北市",
      "confidence": 0.88,
      "needsReview": false
    }
  },
  "warnings": [
    "部分欄位需要確認"
  ]
}
```

### `POST /api/id-card/confirm`

用途：送出使用者確認後的欄位資料。

Request：

```json
{
  "name": "王小明",
  "nationalId": "A123456789",
  "birthDate": "1990-01-01",
  "issueDate": "2020-05-20",
  "issueLocation": "臺北市"
}
```

Response：

```json
{
  "success": true
}
```

API 欄位命名只作工程介面草案；實際 UI 不顯示這些欄位名稱。

## 錯誤處理

相機錯誤：

- 使用者拒絕相機權限時，提示可重新開啟權限或改用其他流程。
- 裝置無可用相機時，提示使用其他裝置。
- 瀏覽器不支援必要 API 時，提示更換支援的瀏覽器。

照片品質錯誤：

- 影像過暗時，提示移至光線充足處。
- 影像模糊時，提示重新拍攝。
- 身分證未對齊時，提示放入框線內。
- 反光嚴重時，提示調整角度。

OCR 錯誤：

- OCR 服務暫時不可用時，提示稍後再試。
- OCR 無法辨識必要欄位時，提示重拍或手動輸入。
- 回傳欄位低信心時，標示欄位需確認。

資料驗證錯誤：

- 身分證字號格式錯誤時，提示確認字號。
- 檢查碼不符時，提示確認字號。
- 日期無法解析時，提示確認日期。
- 必填欄位缺漏時，提示補齊後再送出。

## 測試計畫

文件檢查：

- `docs/plan.md` 使用 UTF-8 無 BOM。
- 文件可正常讀回，沒有替代字元、方框字元、異常問號或亂碼。
- Markdown 標題層級清楚。
- 文件沒有未完成的佔位內容。

前端測試：

- 相機權限允許、拒絕與不可用狀態。
- 手機直向與桌機寬螢幕版面。
- 拍照、重拍、送出辨識流程。
- 低光、模糊、未對齊時的提示。
- 確認頁欄位可編輯與驗證狀態顯示。

後端測試：

- 影像大小限制。
- OCR Provider 成功、逾時與失敗。
- 欄位解析成功與缺漏。
- 身分證字號格式、性別碼與檢查碼。
- 民國日期轉換與不可解析日期。
- 不記錄敏感資料到日誌。

端到端測試：

- 使用清晰身分證正面影像完成辨識與確認送出。
- 使用模糊影像時要求重拍或人工確認。
- OCR 低信心欄位可由使用者修正後送出。
- 後端服務失敗時前端顯示可理解錯誤。
